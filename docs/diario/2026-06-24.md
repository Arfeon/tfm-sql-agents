# 2026-06-24

## Objetivos del día

Montar la infraestructura base del proyecto y verificar que todo arranca correctamente.

## Qué he hecho

He levantado el entorno con Docker Compose: PostgreSQL (con pgvector activado) y Neo4j corriendo en local sin intervención manual. También he cargado el dataset Arcadia directamente al arrancar el contenedor, así cada vez que hago `docker compose up` el entorno ya está listo para trabajar.

Con la infra en pie, he creado una suite de tests diagnóstico en Vitest que valida el estado del sistema antes de empezar a desarrollar agentes:

- que el servidor Postgres responde
- que las dos bases de datos (`arcadia` y `graphsql_memory`) existen
- que pgvector está activo en `arcadia`
- que la conexión a `arcadia` es de solo lectura (cualquier `INSERT` falla con error `25006`)
- que el esquema del dataset tiene las 16 tablas esperadas y los conteos de filas correctos para `seed=42`
- que los datos no tienen anomalías (ratings fuera de rango, sesiones con duración negativa, juegos sin desarrollador)

## Para qué sirve dentro del proyecto

Esta suite es el primer nivel de seguridad del proyecto: si los tests diagnóstico pasan, sé que el entorno está bien montado y puedo empezar a construir agentes sin perder tiempo depurando problemas de infraestructura. También fija el contrato del puerto `ITargetDatabase`, que es la abstracción que usarán todos los agentes para consultar la base de datos objetivo sin acoplarse al driver `pg`.
