# 2026-06-25

## Objetivos del día

Poder crear un modelo LLM configurable —OpenAI en la nube o un modelo local de LM Studio— detrás de una única abstracción, y validar con una pregunta real que la conexión funciona.

## Qué he hecho

He añadido el proveedor LLM siguiendo el mismo patrón puerto/adaptador que ya usé para la base de datos:

- He definido el puerto `IChatModel` (`chat(messages) → texto`): la única interfaz que conocerán los agentes para hablar con el modelo.
- He listado los proveedores disponibles en un `enum LlmProvider` (`OpenAI`, `Local`), para tener en un único sitio la lista cerrada de lo que soporto en vez de cadenas sueltas repartidas por el código.
- He creado un **factory** (`ChatModelFactory`) que, según `LLM_PROVIDER`, crea el adaptador de OpenAI o el local. La clave: elijo el proveedor y construyo **solo ese** (cada adaptador tiene su `fromEnv()` que lee su propia config, igual que `PostgresTargetDatabase.fromParams`), nunca los dos a la vez. Centraliza la decisión y mantiene a los agentes desacoplados del proveedor.
- He implementado dos adaptadores separados, `OpenAIChatModel` y `LocalChatModel`. Descubrí que LM Studio expone una API compatible con OpenAI, así que ambos reutilizan el mismo cliente `ChatOpenAI` de LangChain (el que también usaré al orquestar con LangGraph): el local solo cambia el `baseURL`. Los dejo como clases distintas para que el patrón quede explícito y para poder meter mañana un proveedor no compatible sin tocar nada más.
- He añadido las claves al `.env` y al `.env.example`: `LLM_PROVIDER`, `LMSTUDIO_BASE_URL`, `LMSTUDIO_MODEL`, `LMSTUDIO_API_KEY` y `LLM_TEMPERATURE` (vacía por defecto, porque la familia gpt-5 solo acepta la temperatura por defecto).

He cubierto el componente con tests en dos niveles:

- **Unitarios** (siempre verdes, sin red): si pido `openai` me crea el adaptador de OpenAI, si pido `local` el local, y un proveedor desconocido lanza error. También que la config se resuelve bien desde el entorno.
- **Smoke test de integración**, separado en `npm run test:integration`: dispara un «Hola, dime hola» contra el LLM real y comprueba que responde. Es opt-in y se salta si faltan credenciales, así `npm test` queda siempre verde y offline sin gastar API. Además imprime por consola la respuesta real del modelo, para poder ver lo que contesta al ejecutarlo. Lo he ejecutado contra OpenAI (gpt-5-mini) y responde correctamente («Hola.»).

## Para qué sirve dentro del proyecto

Esto es el segundo cimiento de infraestructura, después de la base de datos: fija el contrato `IChatModel` que usarán todos los agentes (Supervisor, Schema, SQL, Judge…) para razonar. A partir de aquí puedo elegir entre nube y local con una sola variable de entorno, lo que me da flexibilidad para experimentar y para trabajar sin conexión o sin gasto de API durante el desarrollo.
